apps.json

A small, decentralized standard for publishing the apps you make. Put a file at yourdomain.com/apps.json and let people discover, follow, render, and fork the set.

Why this exists

Small apps are starting to behave like posts: frequent, personal, weirdly useful, and scattered everywhere. There is no good way to follow a builder's output as a set — social posts disappear, app stores gatekeep, central directories centralize. apps.json is RSS for apps: one file at a well-known URL, anyone can read, no registry required.

The smallest useful feed

{
  "version": "1.0",
  "apps": [
    { "name": "My Cool App", "url": "https://example.com/cool" }
  ]
}

Required fields are deliberately tiny. Everything else — tags, targets, author, provenance, forkability — is optional creator-declared metadata.

Try the reader

Append your feed URL as a query string:

apps-json.org/?feed=https://yourdomain.com/apps.json
